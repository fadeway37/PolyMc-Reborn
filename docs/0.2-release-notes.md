# PolyMc Reborn 0.2.0-alpha.1+26.1.2 release notes

0.2 is the **Interactive Compatibility Alpha** for exactly Minecraft 26.1.2,
Fabric Loader 0.19.3, and Java 25. It extends the deterministic 0.1 foundation
with explicit interactive projections, stateful full cubes, mapping operations,
and an isolated production playtest architecture. It remains an alpha and does
not promise that arbitrary mods work.

## Implemented engineering scope

### Isolated real-client harness

An independent dedicated server runs the final official-namespace Reborn distribution JAR and
a separate fixture. A real Minecraft 26.1.2 client loads only minimal Fabric
Client GameTest/resource modules plus the driver. The driver rejects Reborn,
Polymer, legacy `polymc`, server fixtures/content definitions, PolyMc-Extra, and
unexpected mods.

The harness defines real multiplayer, resource-pack, movement/look/hotbar,
semantic item use, stateful block, standard-container, explicit entity, and
reconnect scenarios. It writes client/server observations, a loaded-mod list,
summary JSON/Markdown, JUnit XML, logs, and 17 named screenshots to
`build/playtest/`. This is a real client with an automation mod, not a pure
zero-mod vanilla client. On 2026-07-18 all three local entrypoints
`runClientPlaytest`, `runProductionClientPlaytest`, and `runPlaytest` exited
successfully. The retained final bundle records 53/53 checks, 34/34 client
steps, and 17/17 screenshots. GitHub Actions remains a separate gate until its
run ID and downloaded artifact are recorded in the final handoff.

### Safe standard-container GUI

- Explicit initialization-only adapter registry with stable lookup.
- Vanilla generic 9x1 through 9x6 screens and bijective source-slot mapping.
- The real server `Container` remains sole inventory authority.
- Policy-gated pickup, quick move, drag, hotbar/offhand swap, throw, and
  pickup-all; creative clone denied.
- Bounded generation-safe sessions, close/disconnect cleanup, and optional
  strict prediction/resync validation.

This does not implement furnace/progress properties, pagination, custom
buttons/layouts/screens, arbitrary slot subclasses, or automatic projection.

### Explicit virtual entities

- Explicit adapter registry and registered vanilla-surrogate validation.
- Real entity retained with virtual vanilla position/tracking, rotation, name,
  name visibility, and glowing presentation.
- Guarded use/attack callbacks and unload/replacement/server-stop cleanup.

Equipment, passengers, leashes, poses/animations, arbitrary metadata,
dimension specialization, and automatic surrogate selection are not included.

### Stateful full cubes and semantic items

- Canonical per-state keys and deterministic carrier replay/append.
- Bounded unambiguous variant/model/texture resolution and resource hashing.
- Legacy default-state assignment reuse without an unnecessary schema bump.
- Semantic item analysis/use coverage while filtering unsafe client components.

Complex/dynamic geometry, multipart/ambiguous variants, and arbitrary block-
entity renderers remain unsupported or visibly quarantined for protocol safety.
Unsupported/error custom entity and menu types are also marked server-only at
registry freeze; entities receive only an invisible marker quarantine. Their
status remains unsupported and no inferred render/menu compatibility is
claimed.

### Mapping operations

- Stable added/removed/preserved/reassigned/invalidated/resource/capacity diff.
- Read-only status, strict validation, and no-write dry run.
- Atomic checksummed mapping backups and a current-state safety backup.
- Strict restart-only rollback staging and pre-planning activation.

The store remains schema `1`, algorithm `reborn-2`.

## Operator upgrade notes

Back up the world, mod set, and `config/polymc-reborn` before updating. After
startup inspect compatibility, mapping-diff, and resource-pack reports. Use
`/pmcr mappings status`, `validate`, and `diff`; `rollback <id>` only stages a
validated restart-time rollback and never replaces the live plan.

## P1 work deliberately not implemented or not run

- runtime creative reverse mapping;
- pure zero-mod vanilla-client automation;
- completed third-party mod compatibility matrix;
- diagnostic suppression/promotion policy;
- furnace/property GUI and richer entity data/equipment/passengers;
- broad packet transformation.

These stay disabled/fail-closed. A passing internal fixture, when actually
recorded, is not evidence that an unrelated mod is compatible.

PolyMc Reborn is a community-maintained successor derived from PolyMc and is
not an official continuation endorsed by the original authors.
