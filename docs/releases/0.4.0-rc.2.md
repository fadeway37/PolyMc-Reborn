# PolyMc Reborn 0.4.0 RC2 release notes

PolyMc Reborn `0.4.0-rc.2+26.1.2` is a repository-independence and release
infrastructure candidate for exactly Minecraft 26.1.2. Its runtime Java,
Mixin, Access Widener, mapping schema, mapping algorithm, stable API
descriptors, compatibility strategy, and behavioral defaults are unchanged
from RC1.

## Changed in RC2

- Active development moved from the original GitHub fork to an independent
  `fadeway37/PolyMc-Reborn` repository with `main` as its default branch.
- The former fork's branches, pull requests, workflow records, and draft
  releases are retained in the owner-restricted
  [PolyMc-Reborn-Archive](https://github.com/fadeway37/PolyMc-Reborn-Archive).
  Restricting its visibility detached the current GitHub fork-network metadata;
  upstream lineage remains documented through Git ancestry and attribution.
- The project README now serves as a long-term installation, capability,
  compatibility-boundary, and documentation entry point.
- Repository URLs, branch targets, templates, Maven metadata, release artifact
  names, and the candidate version now refer to RC2 and the independent
  repository.
- Public repository checks reject prohibited release-process residue in new
  branches, pull requests, commits, and selected public surfaces without
  rewriting legitimate historical commits.
- Release provenance is rebuilt in the independent repository and binds the
  RC2 commit and exact main Mod JAR, API JAR, SBOM, and checksum manifests.

## Unchanged compatibility and security boundaries

The server continues to keep real modded objects and mechanics authoritative,
with Polymer supplying bounded vanilla-facing representations. Native Polymer
behavior wins by default. Packet fallback and creative reverse mapping remain
disabled; unknown clients remain `VANILLA`; and arbitrary GUI/entity inference,
complex-shape promises, broad packet rewriting, remote compatibility code, and
a mandatory companion client remain outside this release.

PolyMc Reborn does not promise that every mod works. External-mod results are
limited to the exact versions and named features in the inspected release
evidence.

## Verification scope

Repository identity changes invalidate reuse of RC1 hosted provenance as RC2
proof. RC2 therefore repeats unit, API, GameTest, dedicated-server,
single-client, multi-client, external-mod, upgrade, expansion, soak,
cross-platform reproducibility, artifact-content, checksum, and GitHub
provenance gates. The final status and known gaps are recorded in
[the RC2 verification record](../verification/0.4.0-rc.2.md).

The automated client includes minimal Fabric test modules and is not a pure
zero-mod client. Pure zero-mod vanilla automation remains `NOT_RUN` for RC2.
