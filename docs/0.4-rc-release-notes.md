# PolyMc Reborn 0.4.0 RC1 release notes

PolyMc Reborn `0.4.0-rc.1+26.1.2` is the first release candidate for the 0.4
line and targets exactly Minecraft 26.1.2, Fabric Loader 0.19.3, Fabric API
0.155.2+26.1.2, Polymer 0.16.5+26.1.2, and Java 25.

This is a stability release. It preserves the real server-side mod registry
objects and behavior while Polymer supplies bounded client representations.
It does not claim universal Mod, GUI, entity, or complex-block compatibility.

## RC changes

- The cross-platform soak controller no longer stores an open log beneath the
  directory cleaned by nested production playtests. Five-iteration short and
  ten-iteration long modes retain per-iteration process, port, hash, cleanup,
  screenshot, Mod-list, and resource evidence.
- The published 0.3 Beta API signature remains the compatibility input. An
  unchanged Consumer compiled against the exact 0.3 API JAR and an independent
  Consumer resolved from the RC Maven coordinate are separate release gates.
- The external matrix adds Farmer's Delight Refabricated
  `26.1-3.6.7+refabricated` as a server-only food-behavior case alongside
  Immersive Armors and Many More Ores and Crafts.
- Server-only static registry entries use Polymer Registry Sync Manipulator.
  Dynamic registry and recipe-display entries are removed from a client's view
  only when they recursively reference an exact hidden static type. This is
  targeted registry sanitation, not broad packet cancellation.
- The RC release workflow creates GitHub Artifact Attestations for the main
  Mod JAR, API JAR, SBOM, and both checksum manifests, then performs positive
  `gh attestation verify` checks and a tampered-byte negative check.
- The 0.3-to-RC upgrade gate reuses one world and configuration. Mapping schema
  1 and algorithm `reborn-2` are unchanged, so this is compatibility
  validation, not a claimed data migration.

## Security boundaries and known limits

Creative reverse mapping remains disabled and enabling it fails closed.
Packet fallback remains disabled. Unknown clients are VANILLA; Fabric presence
does not authorize registry passthrough. Arbitrary GUI inference, automatic
entity guessing, complex-shape block promises, remote compatibility code, and
a mandatory companion client remain outside this RC.

The automated Client Driver is a real Minecraft 26.1.2 process with only the
minimal Fabric client test modules. It is not a pure zero-Mod vanilla client.
No external content Mod, Reborn, or Polymer is installed on that client.
